# 变量 2.0 State 写入契约

变量 2.0 使用 `<state>...</state>` 块写入状态，使用 `$schema` 定义结构模板。模板不是提示词建议，而是写入守卫：后续写入必须符合已定义的结构、类型和约束。

## 基本格式

```text
<state>
$schema 数据
  背包:
    - 名称: ""
      数量: 0

数据.背包:
  - 名称: 回血药
    数量: 3
</state>
```

上面的 `$schema` 会锁定这些规则：

- `数据` 是对象，只允许 `背包` 字段。
- `数据.背包` 是数组。
- `数据.背包` 的每一项都是对象。
- 每个背包项必须有 `名称` 和 `数量`。
- `名称` 必须是字符串，`数量` 必须是数字。

数组项模板由第一项决定：

```text
背包:
  - 名称: ""
    数量: 0
```

这表示 `背包.0`、`背包.1`、`背包.2` 等每一项都必须遵守同一个对象结构。

## set 是整体替换

`路径: 值` 的 `set` 语义是替换该路径上的完整值，不是对象 merge。

例如：

```text
数据.背包:
  - 名称: 回血药
    数量: 3
```

这会替换整个 `数据.背包` 数组，并递归校验数组里的每一项。

如果只想改某个字段，应该写叶子路径：

```text
数据.背包.0.数量: 4
```

不要用不完整对象去替换数组项：

```text
数据.背包.0:
  数量: 4
```

这会被拒绝，因为整个背包项对象缺少模板字段 `名称`。

## 对象数组会深度校验

对象数组不是只校验数组本身。整体写入根对象、整体写入数组、追加数组项时，都会递归校验数组里的对象。

下面会被拒绝，因为数组项多了 `非法` 字段：

```text
数据.背包:
  - 名称: 回血药
    数量: 3
    非法: x
```

下面也会被拒绝，因为数组项缺少 `数量`：

```text
数据.背包:
  - 名称: 回血药
```

下面会被拒绝，因为 `数量` 不是数字：

```text
数据.背包:
  - 名称: 回血药
    数量: abc
```

数字字符串可以被宽松转换：

```text
数据.背包:
  - 名称: 回血药
    数量: "3"
```

实际写入为：

```json
{
  "背包": [
    { "名称": "回血药", "数量": 3 }
  ]
}
```

## push 也会校验数组项

`+{...}` 会向数组追加一项，追加前会按数组项模板校验。

合法：

```text
数据.背包: +{"名称":"蓝药","数量":"5"}
```

实际追加：

```json
{ "名称": "蓝药", "数量": 5 }
```

非法：

```text
数据.背包: +{"名称":"蓝药","数量":5,"非法":"x"}
```

这会被拒绝，因为数组项不允许模板外字段。

批量追加也会逐项校验。只要其中一项非法，整次 push 会被拒绝：

```text
数据.背包: +[{"名称":"回血药","数量":3},{"名称":"坏药","数量":"abc"}]
```

## 删除与模板字段

模板定义的字段不能删除。

例如 schema 定义了：

```text
$schema 数据
  名称: ""
  等级: 0
```

那么下面会被拒绝：

```text
数据.等级: null
```

如果需要清空某个模板字段，应写入符合类型的空值，例如：

```text
数据.名称: ""
数据.等级: 0
```

## 空对象表示可扩展对象

空对象模板表示该对象内部允许自由扩展：

```text
$schema 数据
  固定: ""
  自由: {}
```

合法：

```text
数据.自由.临时字段: 123
```

非法：

```text
数据.额外: x
```

`自由` 内部可扩展，不代表 `数据` 根对象可以新增模板外字段。

## 动态对象键

如果需要任意名称的对象键，用 `"*"`：

```text
$schema 数据
  同行者:
    "*":
      HP: 0
      状态: ""
```

合法：

```text
数据.同行者.莉娜.HP: 8
数据.同行者.真昼.状态: 休息
```

整体写入也会深度校验每个动态对象：

```text
数据.同行者:
  莉娜:
    HP: 8
    状态: 同行
  真昼:
    HP: 6
    状态: 休息
```

如果某个同行者对象多了模板外字段，写入会被拒绝。

## 约束会递归生效

`$range`、`$step`、`$enum`、`$ro` 等规则会在叶子写入和整体写入时生效。

```text
$schema 数据
  背包:
    - 名称: ""
      数量: 0
$range=[0,10] 数据.背包.[*].数量
```

整体写入时，嵌套的 `数量` 也会被限制：

```text
数据.背包:
  - 名称: 回血药
    数量: 30
```

实际写入为：

```json
{
  "背包": [
    { "名称": "回血药", "数量": 10 }
  ]
}
```

## 推荐写法

初始化或重建一整段结构时，使用整体 `set`：

```text
数据:
  背包:
    - 名称: 回血药
      数量: 3
```

日常局部变化时，优先写叶子路径：

```text
数据.背包.0.数量: 2
数据.同行者.莉娜.HP: -3
```

新增数组项时，使用 `push`：

```text
数据.背包: +{"名称":"蓝药","数量":1}
```

不要用不完整对象替换完整对象，也不要在模板锁定的对象里夹带临时字段。
